# Tutorials Documentation

You are creating hands-on tutorials for grove-meta, the meta-CLI and package manager for the Grove ecosystem.

## Task
Write comprehensive tutorials that:
- Guide users through real-world scenarios
- Build skills progressively from basic to advanced
- Provide hands-on experience with grove-meta features
- Demonstrate best practices and patterns
- Show integration with the broader Grove ecosystem

## Tutorial Topics to Cover
- **Basic Workflow**: Setting up and managing a simple project
- **Multi-Tool Integration**: Using grove-meta with cx, tend, and other tools
- **Workspace Management**: Organizing multiple related projects
- **Tool Installation**: Adding new Grove tools to your environment
- **Custom Workflows**: Creating project-specific automation
- **Team Collaboration**: Sharing Grove configurations
- **Advanced Patterns**: Complex development scenarios

## Structure for Each Tutorial
- **Goal**: What the user will accomplish
- **Prerequisites**: Required knowledge and setup
- **Steps**: Numbered, detailed instructions
- **Explanations**: Why each step matters
- **Verification**: How to check success
- **Troubleshooting**: Common issues and solutions
- **Next Steps**: Where to go from here

## Example Scenarios to Include
- Setting up a new Go project with Grove tools
- Managing multiple microservices in workspaces
- Automating testing with tend integration
- Building and distributing Grove tools
- Creating reproducible development environments
- Migrating existing projects to Grove

## Teaching Principles
- Start with working examples
- Explain concepts through practice
- Build complexity gradually
- Encourage experimentation
- Provide complete, runnable code
- Include expected output

## Output Format
Create well-structured Markdown tutorials with clear objectives, step-by-step instructions, code blocks with syntax highlighting, screenshots or output examples where helpful, and tips for success.
