# Contributing Documentation

You are documenting the contribution guidelines for grove-meta, the meta-CLI and package manager for the Grove ecosystem.

## Task
Write comprehensive contribution documentation that:
- Welcomes new contributors and explains how to get involved
- Provides clear guidelines for different types of contributions
- Explains development setup and workflow
- Describes coding standards and best practices
- Outlines the review and merge process

## Contribution Topics to Cover
- **Getting Started**: How to begin contributing
- **Development Setup**: Setting up the development environment
- **Types of Contributions**: Code, docs, tests, issues, etc.
- **Development Workflow**: Branch strategy, commit conventions
- **Code Standards**: Style guides and quality requirements
- **Testing Requirements**: How to write and run tests
- **Submission Process**: Creating and managing pull requests

## Technical Guidelines to Include
- Go coding conventions and style
- Code organization and package structure
- Error handling patterns
- Testing strategies (unit, integration, e2e)
- Documentation standards
- Performance considerations
- Security best practices

## Process Documentation
- Finding issues to work on
- Creating feature proposals
- Branch naming conventions
- Commit message format
- Pull request template and process
- Code review expectations
- Release process involvement

## Development Tools and Commands
- Building with make
- Running tests with tend
- Code formatting and linting
- Debugging techniques
- Local development tips
- CI/CD pipeline interaction

## Community Guidelines
- Code of conduct expectations
- Communication channels
- Getting help and asking questions
- Reporting bugs effectively
- Suggesting enhancements
- Community meetings or discussions

## Examples to Provide
- Sample pull request workflow
- Good commit message examples
- Test writing patterns
- Documentation update process
- Bug report template usage

## Output Format
Create a welcoming, well-structured Markdown document with clear instructions, practical examples, links to resources, and an encouraging tone that makes contribution accessible to developers of all experience levels.
